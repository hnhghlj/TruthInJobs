## 报告安全漏洞

我们非常重视系统的安全性。如果您发现了安全漏洞，请通过以下方式报告：

### 报告流程

1. **不要公开披露**：请不要在 GitHub Issues 或其他公开渠道发布安全漏洞
2. **发送邮件**：将漏洞详情发送至 security@welfarewatch.com
3. **包含信息**：
   - 漏洞描述
   - 重现步骤
   - 影响范围
   - 建议的修复方案（如果有）
4. **等待响应**：我们会在 48 小时内回复

### 响应时间

- **确认收到**：48 小时内
- **初步评估**：7 天内
- **修复发布**：根据严重程度，14-30 天内

## 安全最佳实践

### 1. 环境变量

**永远不要在代码中硬编码敏感信息：**

```bash
# ✅ 正确：使用环境变量
SECRET_KEY=your-secret-key-here
DB_PASSWORD=your-db-password-here

# ❌ 错误：硬编码在代码中
SECRET_KEY = 'django-insecure-123456'
```

### 2. 密码策略

- 最小长度：8 个字符
- 必须包含：字母和数字
- 使用 Django 内置密码哈希
- 定期提醒用户修改密码

### 3. API 安全

- 使用 HTTPS（生产环境必须）
- 实施请求限流
- 验证所有输入
- 使用 JWT 认证
- 添加 CSRF 保护

### 4. 数据库安全

- 使用参数化查询（ORM）
- 限制数据库用户权限
- 定期备份数据
- 加密敏感字段

### 5. 文件上传安全

- 验证文件类型
- 限制文件大小
- 扫描恶意文件
- 使用独立存储
